# Описание проведенных экспериментов

## Общая постановка

Основной метрикой качества был ROC-AUC

Все основные эксперименты проводились с валидацией через `StratifiedKFold` с числом фолдов `n_splits = 5`. Также пробовал `TimeSeriesSplit` но по итогу стало хуже, либо также поэтому остался на `StratifiedKFold`.

## Проверенные подходы

### 1. Базовый CatBoost на исходных данных

Как база попробвал обучить `CatBoost` по итогу получал около 0.75, признаки никаие для него не генерил.

### 2. CatBoost с генерацией статистических признаков

Далее пробовал CatBoost за счет генерации стат. признаков.
Было реализовано и проверено несколько версий генерации признаков:

- версия `v1`:
  - глобальные статистики по всей истории клиента
  - длина истории клиента `history_len`
  - последние 3 значения признаков
  - последние 3 значения категориальных признаков

- версия `v2`:
  - все признаки из версии `v1`
  - количество уникальных значений категориальных признаков
  - агрегации по последним 3 и 5 записям
  - разности между последним значением и глобальными статистиками
  - разности между последним и предпоследним значением

### 3. Transformer-эмбеддинги + CatBoost

Попробвал Transformer-модель, которая обрабатывала исходные последовательные данные и возвращала эмбеддинги.

Делал так:

1. Обучить Transformer на исходных данных
2. Получить эмбеддинги объектов
3. Сконкатенировать эмбеддинги с исходными признаками
4. Передать получившееся представление в CatBoost

По итогу получил около 0.77.

### 4. Отказ от статистических и исходных признаков

От версии v2 признаков отказался практически сразу по причине: 1) слишком много признаков становилось 2) не особо эффективно 3) слишком много памяти требовало

В последствии отказался вообще от генерации признаков, просто делал последовательности для моделей.

В итоге основной фокус сместился на обучение моделей только на эмбеддингах, которые извлекались из нейросетевых sequence-моделей.

### 5. CatBoost на эмбеддингах xLSTM и Transformer

Обучал catboost уже не на исходных данных, а на эмбеддингах разных нейросетевых моделей:

- xLSTM - по итогу оказалось лучше всех, но училась очень долго
- Transformer
- BiLSTM

Для catboost сначала обучалась сиквенс модель, затем из нее извлекались эмбеддинги, после чего на этих эмбеддингах обучался catboost.


### 6. Замена CatBoost на MLP

Решил попробовать MLP на основе эмбеддингов, чтобы корреляция в мета модели уменьшилась.

Пайплан такой:

1. Сиквенс модель строит эмбеддинг объекта
2. MLP принимает этот эмбеддинг
3. MLP предсказывает финальный скор


### 7. Cross-attention MLP по скрытым состояниям Transformer и BiLSTM

Реализовал модель на основе cross-attention.
Использовал скрытые состояния моделей:
- Transformer
- BiLSTM

## Финальная лучшая модель

Лучшее качество показал ансамбль, построенный как среднее предсказаний трех моделей:

1. `xLSTM embeddings + CatBoost`
2. `BiLSTM + Transformer hidden states + Cross-attention MLP`
3. `BiLSTM embeddings + CatBoost`

В итоге финальный скор **0.78681**

## Борьба с дисбалансом классов

Так как в данных был дисбаланс классов, я пробовал это исправить.

Проверялись следующие варианты в catboost:

```python
auto_class_weights="Balanced"
```

и в нейросетях

```python
scale_pos_weight = (1 - pos_rate) / pos_rate
```

По итогу оказалось слишком агрессивно и сделал чуть мягче

Лучше себя показал более мягкий вариант:

```python
scale_pos_weight = ((1 - pos_rate) / pos_rate) ** alpha
```

где:

```python
alpha = 0.5
```

## Что не успел попробовать

### 1. Модель, обучаемая пол метрику ROC-AUC

Суть в том, чтобы обучать модель не только через `BCE` или `Logloss`, а использовать функцию потерь, которая лучше соответствует ROC-AUC.

```text
score(x_pos) > score(x_neg)
```

То есть положительный объект должен получать скор выше отрицательного.

Можно было попробовать лосс поверх эмбеддингов или предсказаний моделей. Например как-то так:

```python
loss = -log(sigmoid(score_pos - score_neg))
```

### 2. VAE на эмбеддингах моделей

Также была идея сделать VAE

План был таким:

1. Обучить сиквенс модель и получить эмбеддинги
2. Обучить VAE на этих эмбеддингах
3. Использовать VAE для генерации дополнительных эмбеддингов
4. Проверить, помогает ли такая аугментация улучшить финальную модель


### 3. Ортогонализация эмбеддингов

Так как мета модель плохо работало из-за того, что ответы моделй были похоже друг на друга, поэтому была идея попробовать сделать эмбеддинги разных моделей менее дублирующими друг друга.

```python
E_tr = ...    # эмбеддинги Transformer
E_lstm = ...  # эмбеддинги BiLSTM или xLSTM
```

Так как проблема в том, что эмбеддинги разных сиквенс моделей сильно коррелированы, то можно попробовать оставить из `E_lstm` только ту часть, которая не объясняется `E_tr`.

Поэтому хотел попробовать сделать следующее:

- ортогонализацию Gram-Schmidt;
- ridge-регрессию, где `E_lstm` предсказывается по `E_tr`, а затем используются остатки.
